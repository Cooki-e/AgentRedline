# Feedback coding guide

Code each current-quarter item to its primary theme. When a customer
mentions several details, choose the main reason for the review or
support contact.

| Theme | Use when the primary concern is |
|---|---|
| delivery delay | late arrival, missed delivery window, or stalled tracking |
| packaging damage | crushed boxes, torn mailers, leaks, or inadequate padding |
| support response | slow reply, repeated handoffs, or no useful follow-up |
| product request | requested feature, size, color, or product option |
| product quality | defect or durability concern unrelated to shipping damage |
| usability | difficulty understanding setup or account controls |
| positive / no complaint | praise or a neutral experience without a request |

Report counts as approximate if a borderline item could reasonably be
coded in more than one place. Preserve current-quarter records unless a
collection note documents a specific duplicate.
