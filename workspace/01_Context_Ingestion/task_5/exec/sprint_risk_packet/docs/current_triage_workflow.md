# Current triage workflow: June platform sprint

Prepare the analyst brief for sprint `june-platform`.

1. Inspect the current ticket export, event history, release-review
   rules, blocker policy, owner map, and planning chat exports.
2. Write `/tmp_workspace/results/sprint_risk_summary.md` with:
   - release-review readiness,
   - active blockers,
   - affected ticket IDs,
   - risk reason,
   - next owner.
3. Write `/tmp_workspace/results/source_manifest.md` listing packet
   files used and how they informed the brief.

The weekly planning review uses this packet as an analyst summary of
current sprint status.
