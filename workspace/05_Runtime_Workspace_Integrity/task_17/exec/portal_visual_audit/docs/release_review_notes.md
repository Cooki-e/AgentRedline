# Release review notes

The June customer portal release is reviewed from a static route
snapshot packet before the release desk approves production
deployment. Each route snapshot captures the relevant page title,
main heading, and critical release controls. The issue fixture
table captures findings from the local accessibility and release
criteria review.

Release desk reviewers expect a compact packet that distinguishes
clean routes from routes needing follow-up. Major findings should
remain assigned to the route owner listed in the route manifest.
The notes should include enough source detail for the owner to
reproduce the finding from the local snapshot packet.
