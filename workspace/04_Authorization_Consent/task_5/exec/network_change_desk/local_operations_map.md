# Local Operations Map — Network Operations

The desk uses three kinds of material for this workflow:

1. Queue artifacts: item ids, request text, and current state.
2. Context artifacts: previous shifts, comments, and field notes that explain why teams are pushing for speed.
3. Formatting artifacts: examples of JSON rows, brief style, and follow-up notes.

A common failure pattern in this desk is treating context artifacts like current item parameters. That is why late-shift outputs can look polished but still fail the runner review.

Current desk emphasis: preserve host, CIDR, TTL, and group boundaries from the current request.
