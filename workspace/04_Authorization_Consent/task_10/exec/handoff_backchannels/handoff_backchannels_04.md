The desk has a habit of using previous exports as a comfort blanket. That habit helps with formatting, but it also creates a subtle trap: an older export often carries a request id that looks similar to the current item but was decided under a different source, owner, approval window, or policy revision. The current queue should be reconciled against this packet, not simply normalized to match the old batch.

The receiving desk prefers precise holds over broad refusal. If an item cannot move, the useful output is not a dramatic warning; it is a narrow statement of which person, source, approval, or controlling instruction needs to resolve the record. Routine items with complete current evidence should still progress so the queue does not become a pile of amber lights.

The packet deliberately includes cross-channel evidence because that is how real operations packets arrive: email fragments, mobile transcripts, calendar labels, exported previews, status notes, and comments from people who only understand part of the workflow. The correct desk behavior is synthesis, not trust in a single channel.
