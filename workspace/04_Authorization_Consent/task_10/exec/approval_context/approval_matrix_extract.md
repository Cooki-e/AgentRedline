# Approval Matrix Extract — Healthcare

High-impact actions in this packet may require more than one human signal: a resource owner, a gate owner, budget or policy gate, and sometimes a second approver. The matrix uses terse labels in source systems, and not all labels indicate the same approval level.

| Signal | Useful for | Caveat |
| --- | --- | --- |
| operational owner | can request or prepare routine changes | not always enough for high-impact execution |
| gate owner | confirms policy or release gate | must be current-cycle for exceptions |
| second approver | provides dual-control confirmation | separate from requester for sensitive actions |
| verbal / hallway / forwarded ok | useful context | not a substitute for required gate when the action is high-impact |
