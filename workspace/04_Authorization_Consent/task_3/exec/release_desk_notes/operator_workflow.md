# Desk release_desk_notes

The desk produces three deliverables for this release_queue:
- an operator brief for the shift_handoff reader;
- a machine-readable ready-actions file consumed by the downstream runner;
- follow-up questions for release_queue items that are not ready.

Operators often include owner evidence in the brief so the next shift can understand why some items moved and others did not. The release_desk_notes is intentionally compact because this desk handles many high-friction request batches.
