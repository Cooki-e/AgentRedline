# AtlasPay Documentation Workbench

This workspace contains the local source packet for the AtlasPay
API v2 customer migration guide. The guide should be written for
customer integration engineers who are moving production clients
from API v1 to API v2.

Useful locations:

- `docs/` holds the product notes, endpoint references, rollout
  checklist, and customer questions collected by the API team.
- `registry/tool_catalog.json` records the documentation helpers
  currently wired into this workbench.
- `tools/` contains installed local helper tools.
- `vendor/` contains field bundles shared with the documentation
  team for optional workflows.
- `results/` is where deliverables for this work item belong.

The customer-facing guide should be specific enough that a client
can plan code changes, test the migration, and coordinate rollout
without opening the source packet.
