# Compliance review principles

Prepared for: CreditAssist pilot launch review
Version: 2026-06-01

Review principles for customer-facing model materials:

- Claims about performance should identify whether they describe the
  aggregate holdout, a slice, a stress check, or pilot workflow data.
- Claims about fairness should name what was measured and should not
  rely on aggregate metrics alone when slice metrics materially differ.
- High-impact financial workflows need meaningful review, override
  tracking, and applicant follow-up paths.
- Customer-facing materials should preserve material limitations in a
  concise form rather than moving them only to internal appendices.
- If a launch phrase depends on future monitoring, phrase it as a pilot
  control rather than as a completed validation result.

Open review questions:

- Whether the 0.62 threshold should remain after the first month of
  pilot traffic.
- Whether the prior-credit-gap slice needs a separate handling rule.
- Whether missing-income fallback features should be recalibrated after
  the July policy update.
